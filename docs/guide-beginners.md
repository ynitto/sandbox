## 1. Agent Skills とは？

AI エージェント（GitHub Copilot や Claude Code）に **「スキル」という形でノウハウを教える仕組み** です。

### たとえ話

AI エージェントは「何でもそこそこできる新人」のようなものです。
Agent Skills は、その新人に **業務マニュアル（SKILL.md）** を渡して、特定の仕事を高品質にこなせるようにする仕組みです。

### 具体的にできること

| やりたいこと | 対応するスキル | 説明 |
|---|---|---|
| React の画面を作りたい | `react-frontend-coder` | React + TypeScript で3スプリントに分けて実装 |
| API を設計したい | `api-designer` | REST / GraphQL の設計を支援 |
| コードレビューしてほしい | `code-reviewer` | セキュリティ・パフォーマンスの観点でレビュー |
| ドキュメントを書きたい | `technical-writer` | README やガイドを読者中心で作成 |
| 特許明細書を作りたい | `patent-writer` | JPO 様式準拠の特許明細書ドラフトを作成 |
| 複雑なタスクを段階的に進めたい | `scrum-master` | タスクを自動分解してスプリント実行 |

## 2. インストール

### 前提条件

- **GitHub Copilot**（VS Code 拡張、macOS / Windows）または **Claude Code** が利用可能であること
- **Git** がインストール・認証設定済みであること
- **Python 3.10 以上** がインストールされていること

### 手順

```bash
# 1. リポジトリをクローン（URL は実際のリポジトリに合わせてください）
git clone https://github.com/<org>/<repo>.git

# 2. インストールスクリプトを実行
python <repo>/install.py
```

これで以下が自動的に行われます：

1. `~/.copilot/skills/` にコアスキルがコピーされる
2. `~/.copilot/skill-registry.json` にレジストリが作成される
3. ソースリポジトリが自動登録される

> **Windows の場合**: `~/.copilot/` は `%USERPROFILE%\.copilot\` に読み替えてください。

### インストール後の確認

インストールが成功すると、以下のようなメッセージが表示されます：

```
==================================================
インストール完了: 10 件のコアスキル
==================================================

スキル:     /home/user/.copilot/skills
レジストリ: /home/user/.copilot/skill-registry.json

次のステップ:
  - 「スキルをpullして」で最新スキルを取得
  - 「スクラムして」でscrum-masterを起動
  - 「スキルを探して」でリポジトリ内のスキルを検索
```

## 3. スキルの基本的な使い方

Agent Skills は **AI エージェントとの対話で使います**。特別なコマンドは不要で、自然な日本語で依頼するだけです。

### 3-1. スキルを最新の状態にする

```
「スキルをpullして」
```

リポジトリから最新のスキルを取得します。初回インストール後や、チームがスキルを更新した後に実行してください。

### 3-2. スキルを検索する

```
「converter で検索して」        ← インストール済みスキルからキーワード検索
「最新のスキルを検索して」       ← リモートからインデックスを更新して検索
```

### 3-3. 個別のスキルを使う

エージェントに直接依頼するだけで、適切なスキルが自動的に使われます。

```
「REST API のエンドポイントを設計して」       → api-designer が起動
「このコードをレビューして」                   → code-reviewer が起動
「React でログイン画面を作って」               → react-frontend-coder が起動
「このバグの原因を調べて」                     → systematic-debugging が起動
```

### 3-4. フィードバックを記録する

スキルの実行が終わった後、結果に対するフィードバックを記録できます。
これはスキルの改善に役立ちます。

```
「react-frontend-coder に ok を付けて」              ← うまくいった
「code-reviewer に needs-improvement を記録して」     ← 改善の余地あり
```

> **ヒント**: scrum-master 経由でスキルを使った場合は、スプリント終了時に自動でフィードバックが収集されるので、個別の記録は不要です。

## 4. scrum-master でタスクを実行する

`scrum-master` は、複雑なタスクを自動的に分解してスプリント形式で進めてくれるオーケストレーターです。初心者にとって **最もおすすめの使い方** です。

### 発動方法

以下のようなフレーズで依頼するだけです：

```
「スクラムして」
「タスク分解して実行して」
「React + TypeScript でダッシュボードを作って、テストも書いて」
```

### 実行の流れ

```
① スキル探索    → 使えるスキルを自動発見
② バックログ作成 → やるべきことをタスクに分解
③ スプリント計画 → 3〜5タスクを選出（ここでユーザーに確認）
④ タスク実行    → サブエージェントが並列で作業
⑤ レビュー      → 品質確認とフィードバック収集
⑥ 次スプリント  → 残りがあれば③に戻る
```

### 使い方の例

```
ユーザー: 「Todo アプリを React で作って、テストも書いて」

scrum-master の動き:
  Sprint 1: コンポーネント設計 + 基本実装（3タスク）
    → ユーザー承認 → 実行 → レビュー
  Sprint 2: テスト作成 + スタイリング（3タスク）
    → ユーザー承認 → 実行 → レビュー
  最終レポート
```

### 大事なポイント

- **各スプリントの開始前に必ず確認が入る** ので、意図しない作業が勝手に進むことはありません
- 1スプリント = 3〜5タスク、最大5スプリントまで
- 途中で方向修正したい場合は、スプリント承認時に伝えればOK

## 6. よくある質問（FAQ）

### Q: スキルは自動で更新されますか？

はい。エージェントのセッション開始時に自動更新チェックが走ります。手動で更新したい場合は `「スキルをpullして」` と依頼してください。

### Q: スキルを使うのに特別なコマンドを覚える必要がありますか？

いいえ。自然な日本語で依頼すれば、エージェントが適切なスキルを自動で選択します。

### Q: macOS と Windows のどちらでも使えますか？

はい。GitHub Copilot（VS Code 拡張）で利用できます。スキルの保存先は macOS / Linux では `~/.copilot/`、Windows では `%USERPROFILE%\.copilot\` になります。

### Q: インストール後にスキルが認識されない場合は？

VS Code を再起動するか、新しいチャットセッションを開始してください。

### Q: scrum-master が暴走したらどうすればいい？

各スプリントの開始前に必ずユーザー承認が入るので、承認しなければ停止します。また、スプリント上限は5回に設定されています。
