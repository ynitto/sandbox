# Agent Dashboard プロジェクト設定タブ設計

## 背景

プロジェクト設定は選択中プロジェクトだけに適用されるが、現在はプロジェクトヘッダー右端の独立したボタンからダイアログを開く。ボタンがナビゲーションから離れているため、画面全体の設定なのか、選択中プロジェクトの設定なのかが分かりにくい。

## 採用設計

プロジェクト設定を、選択中プロジェクトの内容を切り替えるメインタブへ移す。日常操作から少し距離を置くため、「概要」「タスク」などの後ろに区切りを置き、右端へ「プロジェクト設定」を配置する。その後ろには適用範囲の異なる「共通／全体設定」を置く。

```text
概要　タスク　要対応　実行　成果　定常業務　│ プロジェクト設定 │ 共通　全体設定
```

## ページ構成

- ページ上部に「選択中のプロジェクトに適用」「プロジェクト設定」と説明を表示する。
- 「プロジェクト定義」に憲章、運用ルール、プロジェクトルール、リポジトリの編集導線をまとめる。
- 「調査と高度な設定」は通常操作から分離したカードとして表示し、既存の詳細情報ダイアログを開く。
- 「危険な操作」はページ最下部に置き、リセット操作を通常の設定から視覚的に分離する。
- ファイル編集、詳細情報、リセット確認は一時的な操作なので既存ダイアログを維持する。

## 状態と適用範囲

- プロジェクト未選択時はタブを無効化または非表示にする。
- プロジェクトを切り替えた場合、ページ内容を新しい選択へ追従させる。
- プロジェクト設定ページではプロジェクトヘッダーを残し、対象名を常に確認できるようにする。
- プロジェクトヘッダーは全体設定を含むすべてのタブで維持し、タブ切替時の視線移動を防ぐ。全体設定の適用範囲はページ内の見出しで区別する。
- 設定データ、IPC、編集・リセット処理は変更しない。

## アクセシビリティとレスポンシブ

- プロジェクト設定は既存タブと同じキーボード順序、選択表示、フォーカス表示を使う。
- 日常タブとの区切りは色だけに頼らず、余白と境界線を併用する。
- 狭い画面では既存の横スクロール可能なタブ列に含め、ページ本文には横スクロールを発生させない。
- 主要ボタンは44px相当の操作領域を保つ。

## テスト

- ヘッダー右端のプロジェクト設定ボタンと設定ダイアログが撤去されていること。
- 「プロジェクト設定」タブと対応するページが存在すること。
- 選択中プロジェクト名、4つの定義編集、詳細情報、リセットの導線が表示されること。
- プロジェクト切替後に設定ページが新しい内容へ更新されること。
- 既存のファイル編集、詳細情報、リセット処理が維持されること。
- 全体設定を含むすべてのタブでプロジェクトヘッダーを維持すること。

## 既存スキルとの関係

- `ui-designer`: 適用範囲を示すナビゲーション、フォーカス表示、レスポンシブ配置に利用する。
- `webapp-testing`: 実装後のタブ切替と既存ダイアログへの遷移確認に利用できる。
- `react-frontend-coder`: 対象がプレーンElectronのため使用しない。
