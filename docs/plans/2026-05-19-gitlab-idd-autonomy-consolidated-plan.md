## 2. 変更リスクの評価軸

各施策のリスクは次の 4 軸で判定した。

| 軸 | 内容 | 高リスク側 |
|----|------|-----------|
| **変更範囲** | doc のみ / スクリプト / データモデル・スキーマ | データモデル変更 |
| **既存原則との衝突** | 行動原則・設計思想と整合するか | 「一気通貫」等の最上位原則に抵触 |
| **振る舞い・コスト影響** | スループット・レイテンシ・トークンコストへの影響 | 全イシューに恒常コスト増 |
| **移行・後方互換** | 既存イシュー・記憶・ラベルへの影響 | 移行作業や非互換を伴う |

### 高リスク施策の詳細

- **II-1（設計レビュー関門）**: 新フェーズ追加で worker-role の構造が変わる。行動原則2
  「一気通貫」と正面衝突（分岐①）。全イシューにレビュー往復のコスト・レイテンシが乗る。
  確信度ゲートで軽減するが、ゲート自体が新規。**最も重い施策。**
- **III-1（仕様の共同ドラフト化）**: requester と worker の role 境界（誰が AC を所有するか）を
  再定義する（分岐③）。provisional フラグは MR の新しい状態。AC の living-spec 矛盾を
  解消する点はプラスだが、複数 reference の同時改訂を伴う。
- **IV-3（環境固有知見スコープ）**: ltm-use のコア（memory-format・save/recall/promote）に
  新スコープを追加するデータモデル変更。既存記憶の移行が必要。ブレスト元でも「高コスト」と
  明記済み。単独で大きなプロジェクトになる。

### 低リスク施策（設計判断・移行不要）

I-1 / I-3 / III-2 と V-1 の一部（eval.json・knowledge 使用ログ・KPI）は、doc・テンプレート・
観測の追加のみで既存原則と衝突せず、いつでも巻き戻せる。

## 3. 計画案（パッケージ）

施策を選択可能なパッケージに束ねた。**施策単位での組み替えも可能。**

### 案A — 即効パッケージ（設計判断不要・低リスク・可逆）

- **含む**: I-1, I-3, III-2, V-1（観測系: eval.json・knowledge 使用ログ・KPI）
- **特徴**: すべて doc / テンプレート / 観測の追加。既存原則と衝突せず、いつでも巻き戻せる。
  分岐点の議論を待たずに先行できる。
- **ねらい**: 知識参照の頻度を上げ、記録を高信号化し、証跡を残す。観測を入れて
  以降の施策の効果測定を可能にする。

### 案B — 学習ループ完成パッケージ

- **含む**: 案A + I-2
- **特徴**: 案A に save→recall→rate のクローズとレトロ即時化を足し、「記憶はあるが学習がない」
  状態を解消する。第1観点の課題①（クエリ頻度）の根治。I-2 は rate 自動適用の誤発火
  （正当な記憶を差し戻しで不当に減点）に注意が要る（中リスク）。
- **ねらい**: 記憶を「貯める」から「学習する」へ。

### 案C — 思慮の構造化パッケージ（中核・要設計判断）

- **含む**: II-1, II-2, II-3, III-1
- **特徴**: 当初の不満（思慮が浅い・迎合・振り返りなし・やり直しなし）の中核に直接効く。
  ただし II-1 / II-2 / III-1 は「一気通貫」を最上位から降ろす設計判断（分岐①）が前提。
  II-3（実行検証）のみ判断不要で先行可能。高効果・高コスト。
- **ねらい**: 立ち止まり・複数視点・実行検証を構造として組み込む。

### 案D — 分散実行健全化パッケージ

- **含む**: IV-1, IV-2, IV-3
- **特徴**: gitlab-idd 固有の価値（分散・環境固有知見）を強化。IV-1（分解）と IV-2（継ぎ目封鎖）は
  中リスクで着手しやすい。IV-3 は ltm-use のデータモデル拡張（分岐②）を伴う重量級で、
  単独プロジェクト化を検討する。
- **ねらい**: 分散が誤りを増幅する構造を、検証と知見共有で健全化する。

> 案A ⊂ 案B。案C・案D は案A/B と独立に選択できる。

## 5. 参考: 着手しやすい順序（優先順は人が決める）

断定はしない。依存関係から導かれる「無理のない順序」のみ示す。

1. 分岐点①〜④の議論と **並行して、案A は判断不要なので先行できる。**
2. 案A 投入 → V-1 の観測で現状値（recall 実行率・差し戻し率等）を計測。
3. 案B（I-2）で学習ループを閉じる。
4. 分岐①が「降ろす」で決着 → 案C に着手。決着前でも II-3（実行検証）は先行可。
5. 案D は独立。IV-1/IV-2 は随時、IV-3 は分岐②次第で別プロジェクト化。
