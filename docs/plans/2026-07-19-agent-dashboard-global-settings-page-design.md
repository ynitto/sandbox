# Agent Dashboard 全体設定ページ設計

## 背景

エージェント管理画面とアプリの詳細設定ダイアログは、どちらも選択中のプロジェクトではなく、この端末全体へ適用する設定を扱っている。一方で、別々の導線と表示形式になっているため、設定の所在が分かりにくい。詳細設定を一つのダイアログへ集約すると、項目が縦に連なり、目的の設定を探しにくくなる。

## 採用設計

メイン画面の「エージェント管理」をフルページの「全体設定」へ変更し、既存の通常設定、詳細設定、エージェント管理を同じページへ統合する。

全体設定ページは、上部のコンパクトな分類タブで次の5種類を切り替える。

- **アプリ**: プロジェクトの検索先、自動追加、更新間隔、通知
- **エージェント**: 利用状況、共通指示、利用上限、担当設定、エージェント一覧、画面内AIアシスタント
- **同期と実行**: 自動取得、変更の共有、指示の届け方、共有先
- **定常業務**: 定期実行とステートマシンの実行設定
- **外部連携**: GitLabとレビュー画面の連携

歯車ボタンはダイアログを開かず、「全体設定 > アプリ」へ直接移動する。ダイアログは確認や小規模な一時操作に限定する。

## レイアウト

- 分類インデックスは内容領域の幅を消費しない上部タブとする。
- タブラベルは短い文字列にし、アイコンだけに依存しない。
- 狭い画面では横スクロールや折り返しを発生させず、「設定の種類」セレクトへ切り替える。
- 各分類では既存の見出し、フォーム、カードを再利用し、同時に表示する情報量を抑える。
- エージェント分類内は「利用状況」「共通設定」「エージェント一覧」の既存情報階層を維持する。

## 保存と状態

- 設定は分類単位で保存し、関係のない分類の値を巻き込まない。
- エージェント管理の既存保存操作は各カードに残す。
- 分類を切り替えてもフォームへ入力した未保存値はページを離れるまで保持する。
- 既存の設定データ、IPC、保存先、バリデーションは変更しない。
- 保存失敗は現在表示している分類の操作付近で通知し、既存トーストの動作も維持する。

## アクセシビリティとレスポンシブ

- 分類タブは `tablist` / `tab` / `tabpanel` の関係を持ち、矢印キーとTabキーで移動できる。
- 選択中の分類を文字、コントラスト、`aria-selected` で示す。
- セレクトへ切り替わる画面幅でも、選択中の分類と内容を同期する。
- 操作領域は44px相当、フォーカスリングは既存方針を維持する。
- 375px、768px、1024px、1440px相当で内容の水平スクロールを発生させない。

## テスト

- メインタブとページ見出しが「全体設定」になっていること。
- 歯車ボタンが「アプリ」分類を開くこと。
- 5分類と各設定項目の対応が正しいこと。
- 通常設定・詳細設定ダイアログが撤去され、設定項目が重複しないこと。
- 分類切替、キーボード操作、狭幅用セレクトが同じ状態を共有すること。
- 分類単位の保存が既存設定フィールドだけを正しく保存すること。
- エージェント管理の既存表示・保存・稼働操作が維持されること。

## 既存スキルとの関係

- `ui-designer`: コンパクトな分類ナビゲーション、レスポンシブ、フォーカス表示に利用する。
- `webapp-testing`: 実装後の分類切替、画面幅、主要保存導線の確認と相性がよい。
- `react-frontend-coder`: 対象がプレーンElectronのため使用しない。
