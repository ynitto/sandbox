# Agent Dashboard: ミッション一覧・詳細UI設計

## 背景と目的

ミッション画面は、内部機能の状態を表形式で直接見せており、利用者が状況を理解するために
ロール、ラウンド、オーナーなどの開発用語を読み解く必要がある。

一覧は「何を目指し、今どこまで進み、対応が必要か」を短時間で把握できる構成にする。
担当エージェントの作業状況や会話は、ミッションごとの詳細ダイアログで段階的に開示する。

## 検討した案

| アプローチ | 実装コスト | リスク | 保守性 | 拡張性 | 学習コスト | 推奨度 |
|---|---|---|---|---|---|---|
| 要約カード＋詳細ダイアログ | 中 | 低 | 高 | 高 | 低 | ★★★ |
| 一覧ですべて展開 | 低 | 中 | 中 | 低 | 中 | ★★☆ |
| エージェント別タブ | 高 | 中 | 低 | 高 | 高 | ★☆☆ |

## 採用設計

### ミッション一覧

表形式を要約カードへ変更する。各カードには次の情報だけを常時表示する。

- タイトルと目的
- 利用者向けに翻訳した現在の状態
- 担当状況と進捗の短い要約
- 確認が必要な事項の有無
- 「詳しく見る」操作

内部ID、owner、role、round、bus、agent-amigos などの実装用語は一覧に表示しない。
状態は「担当者を募集中」「作業中」「成果をまとめています」「確認待ち」「完了」など、
利用者が次の展開を予測できる日本語へ変換する。

### ミッション詳細ダイアログ

詳細は次の順序で構成する。

1. **現在の状況** — 目的、進捗、次に起きること、確認が必要な事項
2. **メンバーの作業状況** — 表示名、担当内容、作業中・待機中・完了、短い進捗メモ
3. **やりとり** — 発言を時系列の短い要約で並べ、選択した発言だけ全文を展開

未回答の質問と停止中の担当者はダイアログ上部にまとめ、対応が必要であることを明示する。
エージェントID、メッセージID、ファイルパスなどの技術情報は通常表示しない。

### 会話表示モデル

バス上のメッセージを画面へ直接渡さず、表示専用モデルへ変換する。

- `question` → 質問
- `answer` → 回答
- `request` → 依頼
- `review` → 確認結果
- `decision-request` → 判断のお願い
- `status` / `info` → 進捗共有
- `feedback` → 修正依頼

一覧には件名または本文から作る短い要約を表示し、全文は展開時だけ見せる。
返信関係は親発言の直後へ寄せ、未回答の質問を判別できるようにする。

### データ取得

既存のミッション概要APIを拡張し、次の表示用情報を返す。

- 役割の利用者向け名称と担当内容
- 現在の作業状態、進捗メモ、完了状況
- 重複を除いた時系列メッセージ
- 返信済み・未回答の判定

ファイル配置や内部契約は変更せず、読み取り専用を維持する。

### エラー・空状態

- ミッションが無い場合は、次の行動として「ミッションを依頼」を案内する。
- 会話が無い場合は「まだやりとりはありません」と表示する。
- 一部の状態や会話だけ読めない場合も、取得できた概要は表示する。
- 技術的な読み取りエラーは主表示に露出させず、短い再試行案内と詳細情報へ分離する。

### アクセシビリティとレスポンシブ

- カード全体を曖昧なクリック領域にせず、明示的な「詳しく見る」ボタンを置く。
- 詳細ダイアログには見出し構造、閉じるボタン、可視フォーカスを持たせる。
- 会話全文の展開にはネイティブの `details` / `summary` を利用する。
- 狭い画面ではカードとメンバー一覧を1列にし、横スクロールを発生させない。
- 色だけで状態を表さず、必ず状態名を併記する。

## テスト方針

- ミッション概要が利用者向け状態、担当進捗、会話表示モデルを返すこと。
- inbox と全体チャンネルの同一メッセージを重複表示しないこと。
- 質問と回答の関連、未回答判定、時系列順を確認すること。
- 一覧に内部IDや開発用語が表示されないこと。
- 詳細ダイアログに現在の状況、メンバー、やりとりが表示されること。
- キーボードで詳細を開閉し、発言全文を展開できること。
- 375px、768px、1024px、1440px相当でレイアウトを確認すること。

## 既存スキルとの関係

- `ui-designer`: 段階的開示、カード階層、ダイアログ、アクセシビリティに活用する。
- `webapp-testing`: 実装後の主要導線と画面幅別表示を確認する。
- `react-frontend-coder`: 対象はプレーンElectronのため使用しない。
