## 2. 要旨（3〜5文）

`agent-project` は、単一プロジェクトのバックログを自律的に優先順位付け・実行・検証・収束させ、人の判断が
要る分だけ差し戻す制御層。カレントディレクトリ（または `--root`）をプロジェクトルートとし、
`charter.md` / `repos.json` を入力に成果物を出力する。3層構成（上位＝目標駆動の project ループ、
外側＝backlog 制御の run 正準ループ、内側＝実行委譲先の `agent-flow run`）で、`agent-` 接頭辞は
「実行を agent-flow（既定 kiro-cli）へ委譲すること」を表す。5 つの不変条件（done は verify 終了コード0でのみ確定・
必ず有限回で停止・人の policy がエージェント提案に優先・標準ライブラリのみ・決定的ファイル操作で完結）に
すべての機能が従属する。
