# gitlab-review-viewer

GitLab のイシューと MR を**並べて表示するレビュー専用ビュアー**。
Windows 用の exe としてビルドして起動できる Electron アプリ。

```
┌──────────────┬──────────────────────────┬──────────────────────────┐
│ 検索条件      │ イシュー                  ‖ マージリクエスト          │
│  グループ     │ [Issue #42] [リーダー×]   ‖ [MR !108] [要約×]        │ ← タブ（リーダー/要約も）
│  リポジトリ   │ ☰ URL: https://git...↻⧉↗ ‖ ☰ URL: https://git...↻⧉↗ │ ← ☰ メニュー付き URL バー
│  ラベル       │                          ‖                          │
│  作成者       │   イシューページ          ‖   紐づく MR ページ        │
│  種別/状態    │   （GitLab をそのまま     ‖   （webview 埋め込み）    │
│  キーワード   │     埋め込み表示）        ‖ ← スプリッター（ドラッグ）│
│ ─────────    │                          ‖                          │
│ 候補一覧      ├──────────────────────────┴──────────────────────────┤
│  Issue #42 …  │ 操作対象: Issue #42 — …  opened  status:elaborated   │
│  MR !108 …    │ [コメント本文____________] [承認][差し戻し][却下][変更]│
│  Issue #40 …  │                           [コメント投稿][要約]         │
└──────────────┴──────────────────────────────────────────────────────┘
```

## 機能

- **条件を組み合わせた候補絞り込み** — グループ / リポジトリ（プロジェクト）/
  ラベル（複数・AND）/ 作成者（ユーザー名）/ 種別（イシュー・MR）/ 状態 /
  キーワードをすべて AND で組み合わせて検索し、候補一覧からユーザーが選択する。
  種別（イシューか MR か）は**候補一覧の絞り込みにのみ**効く
- **検索条件のキャッシュ** — グループ / プロジェクトの取得結果と選択値を含め、
  検索条件はすべて前回の入力を保存し、次回起動時に復元する
- **イシュー / MR の並列表示** — 候補を選択すると、紐づくページ（イシューなら
  `related_merge_requests` + `closed_by`、MR なら `closes_issues`）を自動取得し、
  **イシューを左ペイン・MR を右ペイン**に振り分けて表示。複数ある場合は
  各ペインのタブで表示切替できる。イシューに紐づく MR が複数ある場合は、
  イシューとタイトルが同じ MR（`Draft:` / `Resolve "…"` 形式は同一視）の
  タブを自動でアクティブにする。左右ペインの境界は**スプリッターを
  ドラッグしてサイズ変更**できる
- **MR は初回から差分（diff）画面を表示** — MR のタブはページ先頭ではなく
  レビュー対象の **Changes（`/diffs`）** を初期表示する（URL バーから概要等へは
  通常どおり移動できる）
- **MR の事前チェック → 差し戻し確認** — MR を表示したとき（候補選択・MR タブ
  切替）に現在状態を取得し、**コンフリクト**または**未解決（未クローズ）の
  レビューコメント**を検知すると「差し戻すか」の確認ダイアログを出す。
  「差し戻す」を選ぶと検知内容を伝える**固定コメント**を `# 差し戻し` 見出し付きで
  MR に投稿する（ステータスラベルは変えない — ラベル遷移を伴う差し戻しは従来どおり
  アクションバーのボタンで行う）。同じ MR への確認はセッション中 1 回だけ
- **URL バーのメニュー（☰）** — 各ペインの URL 欄のメニューから
  - **リーダーモード** — 表示中ページから本文テキストのみを抽出してタブ表示
  - **要約を作成** — イシュー / MR の内容をエージェントに要約させてタブ表示
  - **Obsidian へ送る** — アクティブなタブの内容を Vault へ書き出す。GitLab ページの
    タブならリーダーモードと同等の本文抽出テキストを送る
- **ローカル CLI エージェントによる要約** — 表示中のイシュー / MR の本文・
  コメント・変更ファイル一覧を指定エージェント（既定は `kiro-cli`）に送って
  要約させる。アクションバーの「要約」はダイアログ表示（コメント欄挿入 /
  Obsidian 送付つき）
- **レビューアクション（承認 / 差し戻し / 却下 / 変更）** — 操作対象
  （表示中のイシュー。イシューが無ければ MR）に対してステータスを進める / 戻す /
  破棄する / 任意に変更する。いずれも入力テキストを本文に `# ボタン名` 見出し付きの
  コメントを対象へ投稿し、実行後は左右ペインのイシュー / MR 表示を自動で
  再読み込みして結果を反映する
  - **承認** — `status:elaborated` → `status:open` に進める。`status:approved` の
    場合はイシューと同タイトルの MR をマージしてイシューをクローズ（コンフリクトや
    未解決レビューコメントがある場合はグレーアウト）。他のステータスはグレーアウト
  - **差し戻し** — `status:elaborated` → `status:draft`、`status:approved` →
    `status:needs-rework` に戻す。他のステータスはグレーアウト
  - **却下** — イシュー名と**似たタイトルのマージリクエストをクローズしてソース
    ブランチを削除**し、イシューは**明示的に閉じる**（表示キャッシュの状態に頼らない。
    既にクローズ済みなら no-op）。**イシューは削除しない** — コメント・経緯が記録として
    残り、委譲元ツール（kiro-flow はイシューのクローズで却下を検知し、人コメントを
    やり直し指示として取り込む）にも決着が正しく伝わる。対象 MR はイシューの
    `related_merge_requests`（open）のうち**タイトル類似度**（タブ選択と同じ
    `titleSimilarity` ≥ 0.5）を満たすものだけ——本文で言及しただけの無関係な MR を
    誤って閉じない。対象はダイアログに事前表示される（MR を直接開いている場合はその MR）
  - **変更** — ラベルプリセット（従来の下ペインと同じ UI）をダイアログで選択し、
    「実行」でステータス変更
- **コメント投稿・ラベルショートカット** — 単独のコメント投稿も可能。`Ctrl+1`〜で
  status ラベルをワンタッチ変更（gitlab-idd スキルのラベル規約が既定）
- **プロキシ設定の引き継ぎ** — `HTTPS_PROXY` / `HTTP_PROXY` / `ALL_PROXY` /
  `NO_PROXY` 環境変数を Chromium に引き継ぎ、埋め込み表示と GitLab API 呼び出しの
  両方に適用する
- **Obsidian エクスポート（オプション）** — 完了したイシュー / MR を要約付きの
  Markdown（frontmatter 付き）にして Vault のフォルダへ書き出す

## セットアップ

```bash
cd tools/gitlab-review-viewer
npm install
npm start          # 開発起動
```

### Windows exe のビルド

Windows 上（または wine 環境）で:

```bash
npm run dist            # release/ に portable exe + NSIS インストーラ
npm run dist:portable   # portable exe のみ
```

`release/GitLab Review Viewer 0.1.0.exe`（portable）をそのまま配布・起動できる。

### 初回設定

1. 起動後、右上の **⚙ 設定** を開く
2. **GitLab URL**（例: `https://gitlab.com` やセルフホスト URL）と
   **アクセストークン**（`api` スコープ）を入力して保存
3. 埋め込みページ側は初回に GitLab へのログインが必要（セッションは
   `persist:gitlab` パーティションに保持され、次回以降は不要）

設定は `%APPDATA%/gitlab-review-viewer/config.json` に保存される。

### プロキシ環境での利用

環境変数にプロキシが設定されていれば自動で引き継ぐ（アプリ側の設定は不要）:

| 環境変数 | 用途 |
|----------|------|
| `HTTPS_PROXY` / `HTTP_PROXY` / `ALL_PROXY` | プロキシサーバ（スキーム別 / 共通） |
| `NO_PROXY` | プロキシを経由しないホスト（カンマ区切り） |

## 使い方

1. サイドバーで条件を設定して **候補を検索**
   - グループ / プロジェクトは 🔍 で検索して選択（未指定なら自分が参加する全体から検索）
   - ラベルはカンマ区切りで複数指定（AND 条件）。プロジェクト選択後は入力補完が効く
   - 作成者はユーザー名（`author_username`）で絞り込む
   - 種別（イシュー / MR）は候補一覧の絞り込みにのみ効く
   - 検索条件（取得済みのグループ / プロジェクト一覧を含む）は自動保存され、
     次回起動時に復元される
2. 候補一覧からイシューまたは MR をクリック → 紐づくイシューが**左ペイン**、
   MR が**右ペイン**にタブ表示される。境界のスプリッターをドラッグすると
   ペイン幅を変更できる
3. 各ペインの URL バー左端の **☰ メニュー**から
   - **リーダーモード** — 表示中ページの本文テキストのみを新しいタブに表示
   - **要約を作成** — エージェントの要約結果を新しいタブに表示
   - **Obsidian へ送る** — アクティブなタブの内容を Vault へ書き出す
   （リーダー / 要約タブは × で閉じられる）
4. 下部のアクションバーで操作。操作対象は**表示されているイシュー**
   （イシューが無ければ表示中の MR）に自動で決まる
   - **承認 / 差し戻し / 却下 / 変更** — 上記レビューアクション。入力テキストが
     `# ボタン名` 見出し付きコメントとして対象へ投稿される
   - **コメント投稿** — 入力テキストをそのまま投稿（`Ctrl+Enter`）
   - **要約** — エージェントに送信し、結果をダイアログ表示。
     「コメント欄へ挿入」「Obsidian へ送る」が可能

## ショートカット（既定・すべて設定でカスタマイズ可能）

| キー | 動作 |
|------|------|
| `Ctrl+1`〜`Ctrl+8` | `status:open` / `blocked` / `in-progress` / `review-ready` / `approved` / `needs-rework` / `needs-clarification` / `done` |
| `Ctrl+Shift+1/2/3` | `priority:high` / `normal` / `low` |
| `Ctrl+Shift+0` | `assignee:any` トグル |
| `Ctrl+Enter` | コメント投稿 |
| `Ctrl+Shift+S` | 要約 |

※ webview（GitLab ページ内）にフォーカスがある間はアプリ側ショートカットは
効かない。アクションバーなどアプリ側 UI をクリックしてから使う。

## ラベルプリセットのカスタマイズ

既定は gitlab-idd スキルのラベル規約（`status:*` / `priority:*` /
`assignee:any`）。「変更」ダイアログの選択肢とキーボードショートカットの両方に
使われる。設定画面の「ラベルプリセット（JSON）」で自由に変更できる:

```json
[
  { "label": "status:approved", "exclusivePrefix": "status:", "shortcut": "Ctrl+5" },
  { "label": "needs-discussion", "toggle": true, "shortcut": "Ctrl+9" }
]
```

- `exclusivePrefix` — 同じ接頭辞の他ラベルを外してから付ける（排他グループ）
- `toggle` — 付いていれば外す / 無ければ付ける
- `shortcut` — `Ctrl+…` / `Ctrl+Shift+…` / `Alt+…` の形式。省略可

## エージェント（要約）の設定

設定画面の「エージェントコマンド」にコマンドテンプレートを指定する。

| プレースホルダ | 意味 |
|----------------|------|
| `{promptFile}` | プロンプト全文を書き出した一時ファイルのパス |
| `{prompt}` | プロンプト全文を argv で渡す（100KB 超は自動でファイル退避 + 参照渡し） |
| （どちらも無し） | 標準入力にプロンプトを流し込む |

既定（kiro-cli）:

```
kiro-cli chat --no-interactive --trust-all-tools "{promptFile} に要約タスクの指示があります。このファイルを読み込み、指示に従って要約だけを出力してください。"
```

他のエージェント例:

```
claude -p "{prompt}"
copilot --no-interactive {promptFile}
```

要約プロンプト本文（`{title}` `{url}` `{state}` `{labels}` `{description}`
`{notes}` `{changes}` `{typeLabel}` が使える）も設定画面で編集できる。

### 高速化と出力フィルター

- プロンプトへ渡す入力は自動で切り詰める（本文 4,000 字・直近コメント 20 件
  ×400 字・変更ファイル 50 件）。省略時はその旨をプロンプトに明記する
- 既定プロンプトは要約を `===SUMMARY_START===` / `===SUMMARY_END===` で
  挟んで出力するよう指示しており、ビュアーはマーカー間の要約本文のみを表示する。
  カスタムプロンプトでも同じマーカーを使うと確実にフィルターされる
- マーカーが無い出力には、スピナー・枠線・進捗表示などの CLI ノイズ行だけを
  取り除く保守的なフォールバックを適用する

## Obsidian エクスポート

設定画面で **Obsidian Vault パス**（と任意のサブフォルダ）を指定すると使える。

- **ペインの ☰ メニュー「Obsidian へ送る」** — アクティブなタブの内容を
  frontmatter（タイトル / URL / 種別 / 状態 / ラベル / 作成者 / 日時）付き
  Markdown として書き出す
  - GitLab ページのタブ → リーダーモードと同等の本文抽出テキスト
  - リーダー / 要約タブ → そのタブの内容
- **要約ダイアログの「Obsidian へ送る」** — 説明本文・変更ファイル一覧付きの
  詳細形式で書き出す

「書き出し後に Obsidian を開く」を有効にすると `obsidian://` URI で
Obsidian が起動する。

## ディープリンク（外部ツールからの起動）

`gitlab-review-viewer://open?url=<GitLab の web_url>` のカスタム URL スキームで、
外部ツールから特定のイシュー / MR をレビュー画面として開ける。
[kiro-projects-viewer](../kiro-projects-viewer/) の GitLab タブ「レビューで開く」が
この入り口を使う。

- シングルインスタンス化されており、起動済みならそのウィンドウで対象を開く
  （未起動なら起動してから開く）
- URL の対象は API で解決され、候補一覧の先頭へ挿入 → 選択（関連 MR / イシューも
  通常の選択と同じように左右のペインへ展開される）
- プロトコル登録は NSIS インストーラ、または起動時の `setAsDefaultProtocolClient` で
  行われる（portable exe は一度起動すれば登録される）

```bash
# 例: コマンドラインから
start "" "gitlab-review-viewer://open?url=https%3A%2F%2Fgitlab.example.com%2Fteam%2Fapp%2F-%2Fissues%2F42"
```

## 構成

```
tools/gitlab-review-viewer/
  package.json          # electron + electron-builder のみ（実行時依存なし）
  src/
    main/
      main.js           # ウィンドウ生成・webview 制御・プロキシ引き継ぎ・ディープリンク
      config.js         # 設定と検索条件キャッシュの読み書き
      gitlab.js         # GitLab REST API v4 クライアント（net.fetch）
      agent.js          # ローカル CLI エージェント実行（要約）
      obsidian.js       # Markdown 生成・Vault への書き出し
      ipc.js            # IPC ハンドラ
    preload.js          # contextBridge（renderer へ安全に API を公開）
    renderer/           # UI（フレームワーク非依存の素の JS/HTML/CSS）
```

## 制限事項

- ペイン内のタブ切替はページを再読み込みする（webview はペインごとに 1 つ）
- 埋め込み表示は GitLab の Web セッション、API 操作はアクセストークンと、
  認証が二系統ある（それぞれ初回に設定 / ログインが必要）
- 候補検索は各種別につき最新 50 件まで
- リーダーモードの本文抽出は表示中ページの DOM（`main` / `article` 相当）からの
  簡易抽出で、動的に読み込まれる部分は表示済みの範囲のみが対象
