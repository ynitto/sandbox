# Как получить токен доступа

Macaroni Messenger работает с git-репозиторием через токен доступа.

Токен нужен, чтобы клиент мог читать и записывать файлы сообщений в выбранный репозиторий.

Если репозиторий публичный, GitHub-адаптер может читать его без токена в read-only режиме. Это подходит для просмотра публичного Macaroni-репозитория, но не для отправки сообщений.

Чтобы писать сообщения, создавать чаты и обновлять `.macaroni/users/<client_id>.json`, нужен токен с правами записи.

Токен не создаёт пользователя. Пользователь в Macaroni - это локальный `CLIENT_ID` и файл `.macaroni/users/<client_id>.json`; токен только открывает доступ к репозиторию. Один общий токен на небольшую доверенную группу технически возможен, но любой человек с этим токеном может менять файлы репозитория.

Важно:

- токен даёт доступ к репозиторию;
- не публикуйте токен в чате;
- не коммитьте токен в git;
- Macaroni Messenger хранит токен в `localStorage`;
- `localStorage` - это удобно, но не банковский сейф.

## GitHub

Это основной пример по умолчанию.

Официальная документация GitHub: [Managing your personal access tokens](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/managing-your-personal-access-tokens).

### Fine-grained personal access token

1. Откройте GitHub.
2. Нажмите на аватар в правом верхнем углу.
3. Откройте **Settings**.
4. В левом меню откройте **Developer settings**.
5. Откройте **Personal access tokens**.
6. Выберите **Fine-grained tokens**.
7. Нажмите **Generate new token**.
8. Укажите имя, например `macaroni-messenger`.
9. В **Repository access** выберите только репозиторий с сообщениями.
10. В **Repository permissions** найдите **Contents** и выберите **Read and write**.
11. Нажмите **Generate token**.
12. Скопируйте токен сразу после создания.
13. Вставьте токен в поле "Токен доступа" в Macaroni Messenger.

GitHub показывает токен только один раз. Если закрыли страницу и не сохранили токен, создавайте новый.

### Какие права нужны

Для MVP нужны права, позволяющие:

- читать файлы репозитория через permission **Contents: Read and write**;
- создавать файлы сообщений через permission **Contents: Read and write**;
- обновлять файлы, если provider API требует update-flow;
- отправлять изменения в репозиторий.

Не выдавайте токену доступ ко всем репозиториям, если можно выбрать один конкретный репозиторий. Fine-grained token как раз хорош тем, что его можно ограничить одним репозиторием и конкретными permissions.

## GitVerse

GitVerse тоже подходит как git-hosting, но browser adapter Macaroni для GitVerse пока не реализован. Если выбрать `GitVerse` в клиенте сейчас, приложение честно покажет ошибку вместо тихого fallback в local test repo.

Плюс в том, что в таком случае оператором ваших персональных данных станет Сбербанк, а это смешно.

Подробная инструкция: [gitverse-token.md](gitverse-token.md).

Коротко:

1. Откройте [GitVerse](https://gitverse.ru/).
2. Войдите в аккаунт.
3. Нажмите на иконку пользователя в правом верхнем углу.
4. Откройте **Настройки**.
5. Перейдите во вкладку **Управление токенами**.
6. Введите имя токена, например `macaroni-messenger`.
7. Отметьте чекбокс **Репозитории**.
8. Нажмите **Генерировать токен**.
9. Скопируйте токен сразу после генерации.

Официальная документация GitVerse: [Токены](https://gitverse.ru/docs/collaborative/authentification/tokens/).

## GitLab И Прочие

Логика получения token та же, но provider adapter должен быть реализован отдельно:

1. Найдите настройки аккаунта.
2. Найдите раздел access tokens / personal access tokens.
3. Создайте токен для одного репозитория или проекта.
4. Выдайте минимальные права на чтение и запись репозитория.
5. Скопируйте токен сразу после создания.
6. Вставьте его в Macaroni Messenger.

Если provider просит выбрать scopes, выбирайте минимальные repository read/write permissions.

## Если Токен Утёк

1. Откройте настройки токенов у provider.
2. Отзовите старый токен.
3. Создайте новый.
4. Обновите токен в Macaroni Messenger.

Git помнит.

Интернет помнит.

Но токен хотя бы можно отозвать.
