# `_first_command_line` パース仕様

## 戻り値と選択順

`_first_command_line(out)` は LLM 応答から **実行可能性のある単一のシェルコマンド行**を抽出し、見つからなければ空文字 `""` を返す。`None` 相当の入力も候補なしとして扱う。

選択順は次で固定する。

1. 応答内のコードフェンスを出現順に走査し、最初に見つかったコマンド行を返す。
2. どのフェンスにもコマンド行がない場合だけ、フェンス外を先頭から走査して最初のコマンド行を返す。
3. 候補がなければ `""` を返す。

したがって、前置きやフェンス外のコマンドが先に現れても、後続フェンス内にコマンドがあればフェンス内を採用する。複数フェンスでは、コマンドを含む最初のフェンスの最初のコマンド行を採用する。

## 入力形状別の受理・拒否

| 入力形状 | 結果 | ルール・根拠 |
|---|---|---|
| `None`、空文字、空白行のみ | `""` | 候補なし |
| `# comment` のみ | `""` | コメントは候補外 |
| `pytest -q` | `pytest -q` | フェンス外の明白なコマンドをフォールバックで受理 |
| 空行・コメント後の `pytest -q` | `pytest -q` | 空行とコメントを無視 |
| `` `pytest -q` `` | `pytest -q` | 単一行 inline code の外側 backtick を除去して受理 |
| `````bash\npytest -q\n````` | `pytest -q` | shell 系言語タグ付きフェンスを受理 |
| `````sh\n# note\n\npytest -q\n````` | `pytest -q` | フェンス内でも空行・コメントを無視 |
| `````\npytest -q\n````` | `pytest -q` | 言語タグなしフェンスを受理 |
| `Here is the command:\n```bash\npytest -q\n```` | `pytest -q` | フェンスを前置き散文より優先 |
| フェンス外 `ruff check .` の後に fenced `pytest -q` | `pytest -q` | 応答上の位置よりフェンス優先 |
| 空またはコメントだけの第1フェンス、コマンドを含む第2フェンス | 第2フェンスのコマンド | 空フェンスを飛ばし、最初の有効な fenced candidate を採用 |
| コマンドを含むフェンスが複数 | 第1フェンスの第1コマンド | 一意で予測可能な出現順 |
| `Here is the command:` のみ | `""` | 英語散文は `sh -n` を通る場合があるため、構文成立だけでは受理しない |
| `I cannot provide a deterministic command.` | `""` | 拒否・説明文は候補外 |
| `$ pytest -q` / `% pytest -q` | `pytest -q` | 一般的な対話シェル prompt を除去して受理 |
| `> pytest -q` | `""` | `>` は shell のリダイレクトと曖昧で、誤変換防止のため prompt として扱わない |
| `1. pytest -q` | `""` | 番号付き散文との区別が不安定なため、リスト記号は正規化しない |
| `python` / `json` 等の非 shell 言語タグ付きフェンス | フェンス内では拒否し、フェンス外へフォールバック | 非 shell コードを verify として誤採用しない |
| 閉じフェンスがない shell 系フェンス | EOF までをフェンスとして走査 | LLM の軽微な Markdown 欠落からコマンドを回収 |
| 閉じフェンスだけ、または fence marker 行 | 候補外 | Markdown 構文はコマンドではない |

shell 系言語タグは大文字小文字を区別せず、空タグ、`sh`、`shell`、`bash`、`zsh`、`console` を受理する。タグの後ろに追加属性がある場合は拒否する。

## 「コマンド行」の判定基準

候補行は ANSI escape を除去し、前後空白を除去してから判定する。空行、Markdown fence marker、先頭が `#` の行は拒否する。inline code は外側の backtick 1 組だけを除去する。`$ ` または `% ` は prompt として除去するが、`>`、箇条書き、番号は除去しない。

正規化後の行は次をすべて満たす場合だけコマンド行とする。

1. 空でなく、改行を含まない単一行である。
2. 日本語を含む全角句読点など、既存 `_PROSE_PUNCT` に該当する散文指標を含まない。
3. `sh -n -c <candidate>` の構文検査に成功する。
4. フェンス外候補ではさらに、先頭 token がコマンド形状である。具体的には shell 予約語、代入、相対・絶対パス、または `pytest`、`python`、`grep`、`test`、`make`、`npm` 等と同じ一般的な実行ファイル token の形を要求し、英語文で典型的な空白区切りの単語列や末尾の英文句読点を拒否する。

フェンス内は shell 系フェンス自体が強い意図表示なので 4 を要求せず、1〜3 で判定する。最終候補は従来どおり `synth_verify` 側の `_looks_like_shell_command` と `_verify_is_degenerate` にも通し、構文不正・恒真式の既存防御を維持する。

## 統合判断

- t1/t5 の「該当テストなし」は現 HEAD と矛盾するため採用しない。現 HEAD には直接テスト2件がある。
- t4 の「テスト1件追加」はコミット実体と一致しないため、仕様根拠には用いない。
- t4 が示した `sh -n` だけでは英語散文を排除できない問題は採用し、フェンス外判定を厳格化した。
- t6 が要求した「前置き＋フェンス」の回帰テスト追加は実装担当への必須受入条件とする。本成果物は仕様確定タスクのため、実装・テスト追加自体は行わない。

## 実装担当の必須受入ケース

最低限、テスト名に `first_command_line` を含め、前置き＋shell fence、タグなし fence、複数 fence、空 fence からの次 fence、非 shell fence、フェンス外散文拒否、prompt 正規化、候補なしを固定すること。
