# `_first_command_line` 現行挙動調査

## 前提・調査範囲

- 「現行が何を返すか」は `tools/kiro-project/kiro-project.py` の `_first_command_line(out)` の直接の戻り値を指すものとして、実際に関数をロードして入力ごとに確認した。
- `synth_verify` への影響も併記する。これは戻り候補を `_looks_like_shell_command` と `_verify_is_degenerate` に通した場合の現行挙動である。
- コマンド例は一貫して `pytest -q`（複数フェンスのみ追加コマンドあり）を用いた。

## 現行ロジックの手追い

各入力行に対して `line.strip()` → `_strip_code()` を行う。`_strip_code` は、行全体がバッククォートで始まり終わる場合に両端を **1 文字ずつだけ**除去する。その後、空文字と `#` 始まりを飛ばし、最初の残った行を返す。フェンス状態、プロンプト、リスト記号は認識しない。

| 形状 | 入力（`\n` は改行） | 手追いと `_first_command_line` の実戻り値 | `synth_verify` への影響 |
|---|---|---|---|
| `bash` フェンス | `` ```bash\npytest -q\n``` `` | 先頭 ` ```bash ` は末尾が `h` なので `_strip_code` で変化せず、即 **` ```bash `** を返す | `sh -n` 不成立で候補棄却。再試行も同じ出力なら最終的に `""` |
| 言語タグなしフェンス | `` ```\npytest -q\n``` `` | 先頭 ` ``` ` の両端を1文字ずつ除去して **`` ` ``** を返す | `sh -n` 不成立で候補棄却。最終的に `""` |
| フェンス前の前置き散文 | `Here is the command:\n```bash\npytest -q\n```` | 最初の非空行なので **`Here is the command:`** を返し、後続フェンスを見ない | 英語散文は全角句読点を含まず `sh -n` も通るため、その散文を verify として誤採用 |
| `$` プロンプト接頭辞 | `$ pytest -q` | 記号を除去せず **`$ pytest -q`** | `sh -n` を通るため接頭辞込みで誤採用 |
| `%` プロンプト接頭辞 | `% pytest -q` | 記号を除去せず **`% pytest -q`** | `sh -n` を通るため接頭辞込みで誤採用 |
| `>` プロンプト接頭辞 | `> pytest -q` | 記号を除去せず **`> pytest -q`** | `sh -n` を通るため接頭辞込みで誤採用（実行時には `pytest` へのリダイレクトとして解釈され得る） |
| 番号付きリスト | `1. pytest -q` | 番号を除去せず **`1. pytest -q`** | `sh -n` を通るため番号込みで誤採用 |
| 複数フェンス | `` ```bash\npytest -q tests/unit\n```\n```bash\nruff check .\n``` `` | 最初のフェンス開始行で止まり **` ```bash `**。第1・第2コマンドとも見ない | 候補棄却、最終的に `""` |
| フェンス内の空行・`#` コメント | `` ```bash\n\n# run focused tests\npytest -q tests/unit\n``` `` | 空行・コメントへ到達する前にフェンス開始行で止まり **` ```bash `** | 候補棄却、最終的に `""`。なおフェンス開始行を除いた入力なら空行とコメントを飛ばしてコマンドを返せる |
| コマンドを含まない散文応答 | `I cannot provide a deterministic command.` | 最初の非空行をそのまま **`I cannot provide a deterministic command.`** | 英語散文は `sh -n` を通るため verify として誤採用 |
| 空行・コメントだけ（コマンドなし） | `\n# no deterministic command\n` | 空行とコメントをすべて飛ばして **`""`** | 候補なしとして再試行し、同じなら最終的に `""` |

## サマリー

取りこぼしの根因は、コードフェンスを状態として解析せず「最初の意味ありそうな行」を返す点にある。フェンス開始行はコマンドの前で誤選択され、前置き散文・プロンプト接頭辞・番号は除去されない。さらに後段の妥当性確認が `sh -n`（構文確認）中心なので、英語散文や接頭辞付き文字列の一部は候補として通過する。

## 検証・未解決事項

- 上表の全入力を現行関数へ渡し、記載した戻り値と後段2判定の結果を実測した。
- 完了条件用に、修正後も維持されるべき「空行とコメントを飛ばし、生コマンドを返す」基準テストを1件追加した。問題挙動そのものは将来の修正を阻害しないようテストへ固定していない。
- 本タスクは列挙・現行挙動記録が担当範囲のため、パーサ実装の修正、プロンプト除去方針、複数フェンス時にどのコマンドを選ぶかの仕様決定は後続タスクへ残した。
- 範囲外で、`_looks_like_shell_command` が構文上成立する英語散文をコマンド扱いする問題も確認した。フェンス抽出修正だけでは、コマンドを含まない英語応答の誤採用は残る。
